# 自适应竞品分析

## 适用场景

你需要分析一个产品的竞争格局，但不同行业的竞品关注点完全不同。
输入你的场景，自动用最合适的框架来分析。

## 输入参数

- `industry`：行业（SAAS / 电商 / 金融 / 内容 / 教育 / 企业服务 / 工具 / 其他）
- `companySize`：公司规模（startup / mid / large）
- `productStage`：产品阶段（0-1 / growth / mature / decline）
- `competitors`：竞品列表（2-5 个竞品名称或描述）
- `focus`：分析重点（默认"全面"；可选"价格""功能""用户体验""增长策略"）

## 自适应逻辑

### 基于行业选择分析框架

如果行业是 **SAAS 或 企业服务**，使用"功能-定价-服务"框架：
- 产品功能矩阵（功能覆盖度、深度、差异化）
- 定价策略对比（订阅模式、免费增值、按量计费、企业定制价）
- 实施与集成成本（部署方式、API 丰富度、第三方集成生态）
- 售后服务（SLA 等级、响应时间、客户成功团队）
- 客户案例与口碑（行业覆盖、客户规模、用户评价）

如果行业是 **电商或内容**，使用"流量-转化-复购"框架：
- 用户体验与交互设计（核心路径流畅度、页面加载、搜索推荐精度）
- 获客渠道与成本（自然流量 vs 付费流量的比例、CPA、LTV/CAC）
- 转化漏斗（浏览量→加购→下单→支付每一步的转化率）
- 复购与留存策略（会员体系、推送机制、优惠策略）
- 供应链与履约能力（仅电商：仓储、物流、退货处理）

如果行业是 **金融**，使用"安全-合规-体验"框架：
- 安全与风控能力
- 合规资质与监管覆盖
- 产品体验与触达渠道
- 费率结构与隐藏成本

如果行业是 **教育或工具**，使用"用户增长-黏性-变现"框架：
- 获客方式与病毒系数
- 用户激活与次日留存
- 核心功能完成率
- 变现路径与客单价

### 基于公司规模调整输出格式

如果公司规模是 **初创（startup）**：
- 输出简洁 1 页摘要
- 聚焦 3 个最关键差异点
- 给出"我们的快速切入机会在哪"

如果公司规模是 **中型（mid）**：
- 输出 2-3 页分析报告
- 含功能对比矩阵 + 定价对比表
- 给出"差异化方向建议"

如果公司规模是 **大型（large）**：
- 输出完整竞品报告（含执行摘要、详细对比、SWOT、战略建议）
- 每个维度打分（1-5 分）
- 给出"三个战略选项（保守/进取/颠覆）"

### 基于产品阶段调整分析深度

如果产品阶段是 **0-1**：
- 重点分析已有竞品的弱点和空白区
- 输出"最小可行差异化点"
- 关注竞品未覆盖的用户需求

如果产品阶段是 **增长期**：
- 重点分析竞品的增长策略和获客渠道
- 输出"可复用的增长杠杆"
- 关注竞品的留存和变现手段

如果产品阶段是 **成熟期**：
- 重点分析竞品的功能演进路线
- 输出"防御策略和新的增长曲线"
- 关注行业格局变化和新进入者威胁

## 使用示例

### 输入
```
industry: SAAS
companySize: mid
productStage: growth
competitors: 竞品A（项目管理工具）, 竞品B（协作平台）, 竞品C（轻量版）
focus: 功能与定价
```

## 设计原则

1. **不是复制黏贴模板** — 行业不同，框架不同，分析维度完全不同
2. **输出长度根据场景自适应** — 初创公司不需要百页报告
3. **可操作** — 每个分析必须给出 actionable 的建议
4. **对比视角** — 永远要回答"所以我们该怎么做"
