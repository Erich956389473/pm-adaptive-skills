# 自适应用户画像

## 适用场景

你需要建立用户画像来指导产品决策。
B端产品的决策链复杂，C端产品的心理动机多样。
输入你的场景，自动生成最合适的画像框架。

## 输入参数

- `industry`：行业（SAAS / 企业服务 / 电商 / 金融 / 内容 / 教育 / 工具 / 其他）
- `companySize`：公司规模（startup / mid / large）
- `productType`：产品类型（B2B / B2C / B2B2C）
- `sourceData`：已有数据（可选：访谈记录 / 问卷数据 / 后台行为数据 / 无数据）
- `useCase`：用途（可选：产品设计 / 营销定位 / 运营策略 / 融资BP）

## 自适应逻辑

### 基于产品类型调整画像维度

如果产品是 **B2B 或 企业服务**：
- 核心维度：公司属性（行业、规模、营收区间）、决策链（使用者/决策者/审批者/预算持有者）、使用场景（日常使用/特殊项目）、采购流程（试用→评估→决策→签约周期）、预算权限、痛点类型（效率/成本/合规/竞争压力）
- 输出格式：每个画像含 company profile + decision chain map + use case + pain points
- 数量建议：初创 2-3 个，大型 5-7 个

如果产品是 **B2C 或 内容/工具**：
- 核心维度：人口属性（年龄、城市、职业）、行为特征（使用时段、频率、核心功能）、心理特征（动机、价值观、审美偏好）、付费意愿、社交影响力（KOL/KOC/普通用户）
- 输出格式：每个画像含 name + photo description + quote + behaviors + needs + frustrations
- 数量建议：初创 1-2 个，大型 4-6 个

如果产品是 **B2B2C**（平台型产品）：
- 同时覆盖两端：B端商家画像 + C端消费者画像
- B端侧重：经营规模、入驻动机、平台依赖度
- C端侧重：使用习惯、信任来源、决策因素

### 基于公司规模调整数量

如果公司规模是 **初创（startup）**：
- 只做 1-2 个核心画像，聚焦"早期 adopters"
- 每个画像含 1 页信息
- 快速验证，不追求完美

如果公司规模是 **中型（mid）**：
- 做 3-4 个画像，覆盖主要用户群
- 每个画像含 2 页，含详细的场景描述
- 可以基于后台数据验证画像准确性

如果公司规模是 **大型（large）**：
- 做 5-7 个画像，含细分场景
- 每个画像含完整 profile + user journey map
- 需要数据支撑（问卷+行为数据+访谈）
- 画像需要定期更新

### 基于数据来源调整可信度标注

- 如果 sourceData 包含访谈记录 → 标注为"定性验证"级别
- 如果 sourceData 包含问卷数据 → 标注为"定量验证"级别
- 如果 sourceData 包含后台行为数据 → 标注为"行为验证"级别
- 如果 sourceData 为"无数据" → 标注为"假设版，仅作讨论用"

## 使用示例

### 输入
```
industry: 企业服务
companySize: mid
productType: B2B
useCase: 产品设计
sourceData: 已有5份客户访谈记录
```

### 输出（节选）

```
## 用户画像：企业服务项目管理工具

### 基于您提供的 5 份客户访谈，建立以下 3 个核心画像

---

### 画像 1：张总 — IT 部门负责人（决策者）
"我需要一个能让老板看到IT价值的工具"

- 公司背景：500-1000 人企业，年营收 5-10 亿
- 决策角色：关键决策者，有预算审批权
- 核心痛点：老板看不到IT部门产出、跨部门协作混乱
- 购买动机：向上汇报需要数据支撑
- 决策周期：3-6 个月
- 关键需求：报表能力、权限管理、企业级安全
- 触达渠道：行业峰会、CIO圈子、咨询公司推荐

### 画像 2：小李 — 研发主管（使用者/推动者）
"我需要一个不添乱的工具"

- 背景：30人研发团队，正在从Excel迁移到专业工具
- 核心痛点：现有工具太复杂，团队成员不愿意用
- 关键需求：上手快、导入方便、移动端可用
- 影响力：会向张总推荐工具，但不决策

### 画像 3：王经理 — 采购（审批者）
"预算有限，ROI要说清楚"

- 关注点：价格透明、无隐藏费用、已有客户案例
- 核心质疑："和竞品A有什么区别？"
- 决策影响：可以否决，不能最终拍板

### 产品设计建议
- 优先满足"小李"（使用者）的体验——他决定工具能不能用起来
- 同时给"张总"（决策者）提供管理视角——他决定买不买
- 不要把"王经理"（采购）逼到墙角，留谈判空间
```
