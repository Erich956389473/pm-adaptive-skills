# 自适应产品路线图

## 适用场景

你需要出产品路线图，但 0-1 的产品和成熟产品的路线图完全是两码事。
初创按周规划，成熟产品按季度规划。
输入你的场景，自动生成最适合的路线图模板和规划粒度。

## 输入参数

- `productStage`：产品阶段（0-1 / growth / mature / decline）
- `companySize`：公司规模（startup / mid / large）
- `quarterlyGoals`：季度目标（可选，1-3 个）
- `currentResources`：可用资源（可选，如：8人团队，含3前端2后端1设计1测试1PM）

## 自适应逻辑

### 基于产品阶段调整时间粒度

如果产品阶段是 **0-1**：
- 时间粒度：按周（Next 4 Weeks / Future）
- 不规划超过 1 个月的内容（变化太快）
- 每周复盘调整
- 输出格式："本周目标 — 本月方向 — 未来不确定"

如果产品阶段是 **增长期**：
- 时间粒度：按月（Next 3 Months / Next 6 Months / Future）
- 按月度里程碑出，季度回顾
- 输出格式：月度主题 + 关键结果 + 信心度标注

如果产品阶段是 **成熟期**：
- 时间粒度：按季度（Q1 / Q2 / Q3 / Q4）
- 年度规划 + 季度调整
- 输出格式：季度主题 + OKR + 项目级任务

### 基于公司规模调整输出格式

如果公司规模是 **初创（startup）**：
- 格式："主题 — 目标 — 关键结果"（单行版）
- 不做层级拆解，几句话说明白
- 一张图就能装下全部

如果公司规模是 **中型（mid）**：
- 格式："主题 — 目标 — 关键结果 — 负责人"
- 含跨团队依赖标注
- 1-2 页文档

如果公司规模是 **大型（large）**：
- 格式："Epic — Feature — Story" 三层级
- 需要标注：状态、优先级、依赖、风险
- 正式文档 + 同步工具（Jira/Notion/飞书）

### 使用示例

### 输入
```
productStage: growth
companySize: mid
quarterlyGoals:
  - 提升月活跃用户30%
  - 扩展企业版功能
  - 降低首月流失率
currentResources: 8人（3前端2后端1设计1测试1PM）
```
