# 自适应数据分析指南

## 适用场景

你需要做数据分析，但不同角色、不同行业看的指标完全不同。
数据PM和普通PM的关注点差十万八千里。
输入你的场景，自动推荐最相关的分析框架和指标。

## 输入参数

- `role`：PM角色（pm / data-pm / growth-pm / strategy-pm）
- `industry`：行业（SAAS / 电商 / 金融 / 内容 / 教育 / 企业服务 / 工具）
- `productStage`：产品阶段（0-1 / growth / mature / decline）
- `goal`：分析目标（可选：发现问题 / 验证假设 / 汇报总结 / 指标搭建 / 监控报表）
- `dataSource`：数据源（可选：SQL自助 / BI工具 / 自动报表 / 无现成数据）

## 自适应逻辑

### 基于 PM 角色调整分析侧重点

如果角色是 **数据PM（data-pm）**：
- 侧重：指标体系搭建、埋点方案设计、数据仓库规划、AB测试设计
- 推荐产出：数据字典、指标口径文档、报表原型、埋点验收清单
- 输出风格：技术细节完整、包含SQL/BI配置参考

如果角色是 **普通PM（pm）**：
- 侧重：自助查数方法、常见指标解读、如何用数据驱动决策
- 推荐产出：看板截图 + 关键洞察 + 行动建议
- 输出风格：避开技术细节，聚焦业务结论

如果角色是 **增长PM（growth-pm）**：
- 侧重：漏斗分析、实验设计、用户分群、渠道归因
- 推荐产出：增长实验报告、漏斗分析、CLV预测
- 输出风格：假设驱动，A/B测试导向

如果角色是 **策略PM（strategy-pm）**：
- 侧重：行业大盘、竞品数据、市场趋势、商业模型验证
- 推荐产出：行业分析报告、TAM/SAM/SOM测算、ROI模型
- 输出风格：宏观视角，财务与业务结合

### 基于行业选择核心指标集

如果行业是 **SAAS**：
- 北极星指标推荐：ARR / NRR / 月活企业数
- AARRR指标：
  - 获客：新注册企业数、CAC、MQL-SQL转化率
  - 激活：激活率、首周核心功能完成率
  - 留存：月活跃率、NDR、Churn Rate
  - 收入：ARR、MRR、LTV、扩展收入占比
  - 推荐：NPS、推荐率

如果行业是 **电商**：
- 北极星指标推荐：GMV / 年购买用户数
- AARRR指标：
  - 获客：新增用户、CPA、渠道ROI
  - 激活：首次购买转化率
  - 留存：复购率、30日/90日留存
  - 收入：GMV、客单价、ARPU
  - 推荐：分享率、口碑传播系数

如果行业是 **内容/社区**：
- 北极星指标推荐：DAU / 内容消费时长
- 核心指标：MAU、日活率、人均使用时长、内容生产量、互动率、完播率

如果行业是 **金融**：
- 北极星指标推荐：管理资产规模(AUM) / 交易量
- 核心指标：获客成本、首次交易转化、客均资产、活跃率、坏账率

### 基于产品阶段推荐指标集复杂度

如果产品阶段是 **0-1**：
- 推荐最小可用指标集：3-5 个北极星候选指标
- 不建完整指标体系，聚焦"什么指标能证明产品价值"
- 埋点策略：关键行为 + 核心转化事件

如果产品阶段是 **增长期**：
- 推荐标准 AARRR 指标集（15-20 个核心指标）
- 建立日报/周报机制
- 引入漏斗分析和分群分析

如果产品阶段是 **成熟期**：
- 推荐完整指标体系（60+ 指标，含过程指标和结果指标）
- 建立异常预警机制
- 引入 LTV 模型和用户分层模型

## 使用示例

### 输入
```
role: pm
industry: SAAS
productStage: growth
goal: 发现用户流失原因
dataSource: BI工具（已有报表）
```

### 输出（节选）

```
## 数据分析指南：SAAS 产品用户流失诊断

### 适配说明
基于您的场景（普通PM / SAAS / 增长期 / 流失诊断），
建议使用"分群对比 + 流失节点分析"方法。

### 一、建议的分析路径
步骤1：定义流失（建议：连续30天未使用核心功能的账号）
步骤2：分群对比（按：注册月份 / 套餐类型 / 行业）
步骤3：定位流失节点（在哪个环节流失率最高）
步骤4：深度分析流失原因（结合用户行为 + 客服记录）

### 二、核心要看的关键指标
| 指标 | 参考值 | 说明 |
|------|--------|------|
| 月流失率 | SAAS平均水平5-8% | 越高越危险 |
| 首月留存 | 好产品应该>40% | 首月是留存关键期 |
| NDR (净留存率) | 好产品>100% | 扩展收入是否覆盖流失 |
| 核心功能使用率 | 与留存相关性最高 | 找到"魔法数字" |

### 三、建议的自助分析SQL模板
```sql
-- 找流失用户的共同特征
SELECT plan_type, industry, signup_month, COUNT(*) as churned_users
FROM users
WHERE last_active < DATE_SUB(NOW(), INTERVAL 30 DAY)
  AND status = 'active_previously'
GROUP BY plan_type, industry, signup_month
ORDER BY churned_users DESC
```

### 四、可落地的行动建议
1. 先看"首月流失"——产品 onboarding 可能有问题
2. 再看"哪个套餐流失率高"——定价或服务问题
3. 找出 3 个流失用户做回访——定性验证定量发现
```
