# PM 自适应技能包

**每个技能，都能自适应你的场景。**

pm-skills 给你了一套固定模板——不管你做 SaaS 还是电商，出来的东西都一样。
这里不一样：**输入你的行业、阶段、团队规模，技能自动调整分析框架和输出粒度。**

## 和 pm-skills 的区别

| | pm-skills | PM 自适应技能包 |
|---|---|---|
| **框架选择** | 固定模板（所有人一样） | **根据行业/阶段动态选择** |
| **输出粒度** | 固定长度 | **根据公司规模调整（初创1页 / 大型完整报告）** |
| **指标推荐** | 通用指标 | **根据行业推荐核心指标（SaaS看ARR，电商看GMV）** |
| **角色适配** | 只面向通用 PM | **区分数据PM/增长PM/策略PM** |
| **语言** | 英文 | **中文** |
| **格式** | Claude 命令 + 技能 | **纯 Skill Markdown，任何 Agent 可用** |

## 技能列表

| 技能 | 自适应维度 | 适用场景 |
|------|-----------|---------|
| **competitive-analysis** | 7行业×3规模×4阶段 → 不同分析框架 | 分析竞品格局 |
| **data-analysis** | 4角色×7行业×3阶段 → 不同指标体系 | 做数据分析决策 |
| **user-persona** | B2B/B2C/B2B2C×3规模 → 不同画像维度 | 建立用户画像 |
| **prioritize** | 4阶段×3规模 → 不同优先级模型 | 需求排期 |
| **roadmap** | 4阶段×3规模 → 不同时间粒度 | 编制产品路线图 |

## 怎么用

**在任何支持 Skill 的 AI Agent 中使用（Claude Code / Kimi CLI / Cursor / Codex）：**

```
告诉我你要分析的产品场景（行业、阶段、团队规模），
然后我自动调用最合适的技能。

例如：
"我是 SaaS 公司增长期 PM，想分析竞品"
→ 自动调 competitive-analysis（SaaS版 + 增长期粒度）

"我是数据PM，刚接手一个电商产品"
→ 自动调 data-analysis（数据PM版 + 电商指标）
```

## 设计原则

1. **不是模板，是框架选择器** — 分析什么、怎么分析，由你的场景决定
2. **深度匹配** — B端和C端的竞品、画像、指标完全不同
3. **输出长度看人下菜碟** — 初创要快，大厂要全
4. **中文优先** — 面向中国 PM
