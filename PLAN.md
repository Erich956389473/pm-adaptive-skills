# PM 自适应技能包 — 验收标准

## 核心逻辑

用户输入产品场景信息（行业、公司规模、产品阶段、PM角色），
技能自动调整分析维度和输出格式。

## 自适应维度表

| 维度 | 取值 | 影响范围 |
|------|------|---------|
| **行业** | SAAS / 电商 / 金融 / 内容 / 教育 / 企业服务 | 竞品框架、数据分析指标、用户画像维度 |
| **产品阶段** | 0-1 / 增长期 / 成熟期 / 衰退转型 | 优先级模型、路线图粒度、指标选择 |
| **公司规模** | 初创(1-10人) / 中型(10-200) / 大型(200+) | 流程复杂度、文档详细度、工具推荐 |
| **PM角色** | 产品经理 / 数据PM / 策略PM / 增长PM | 输出侧重点、分析深度 |

## 第一批技能清单 & 验收条件

### 技能 1：自适应竞品分析（competitive-analysis）

**验收条件：**
- [ ] 输入行业=SAAS 时，分析维度包含：产品功能、定价策略、实施成本、售后服务、API生态
- [ ] 输入行业=电商时，分析维度包含：用户体验、获客成本、物流履约、供应链深度、复购率
- [ ] 输入公司规模=初创时，输出 1 页摘要，聚焦关键差异点
- [ ] 输入公司规模=大型时，输出完整报告模板，含详细对比矩阵
- [ ] 不同行业调用的框架名称不同（SaaS用"功能-定价-服务"框架，电商用"流量-转化-复购"框架）

### 技能 2：自适应数据分析指南（data-analysis）

**验收条件：**
- [ ] 输入角色=数据PM时，侧重：指标体系搭建、埋点方案、报表设计
- [ ] 输入角色=普通PM时，侧重：自助查数、常见指标解读、数据驱动决策案例
- [ ] 输入行业=SAAS时，核心指标包括：ARR、MRR、Churn Rate、LTV、CAC
- [ ] 输入行业=电商时，核心指标包括：GMV、转化率、客单价、复购率、退货率
- [ ] 输入产品阶段=0-1时，推荐最小可用指标集（3-5个北极星候选）
- [ ] 输入产品阶段=成熟期时，推荐完整指标体系（60+指标，按AARRR分类）

### 技能 3：自适应用户画像（user-persona）

**验收条件：**
- [ ] 输入行业=企业服务/SAAS时，画像维度包含：决策链角色、预算权限、采购流程、年度合同额
- [ ] 输入行业=C端时，画像维度包含：使用场景、心理动机、社交影响力、付费意愿
- [ ] 输入公司规模=初创时，输出1-2个核心画像（快速聚焦）
- [ ] 输入公司规模=大型时，输出5-7个细分画像（支持精细化运营）

### 技能 4：自适应需求优先级排序（prioritize）

**验收条件：**
- [ ] 输入产品阶段=0-1时，默认框架为 RICE 简版（只保留 Reach + Impact）
- [ ] 输入产品阶段=成熟期时，默认框架为完整 RICE + 战略对齐
- [ ] 输入公司规模=初创时，推荐"影响-成本"2x2矩阵
- [ ] 输入公司规模=大型时，推荐"影响力-可行性-战略对齐"三维评分
- [ ] 输出包含建议的执行顺序和时间估算

### 技能 5：自适应产品路线图（roadmap）

**验收条件：**
- [ ] 输入产品阶段=0-1时，时间粒度为周（Next 4 weeks / Future）
- [ ] 输入产品阶段=增长期时，时间粒度为月（Next 3 months / Next 6 months / Future）
- [ ] 输入产品阶段=成熟期时，时间粒度为季度（Q1-Q4）
- [ ] 输入公司规模=初创时，输出格式为"主题-目标-关键结果"
- [ ] 输入公司规模=大型时，输出格式为"Epic - Feature - Story"层级

## 格式要求

- [ ] 每个技能独立一个 .md 文件，Agent 可以直接读取调用
- [ ] 自适应逻辑写在 Skill 的 prompt 里，通过"如果...那么..."条件分支实现
- [ ] 所有内容中文
- [ ] 每个技能开头标注：适用场景、输入参数、输出格式
- [ ] 提供 1 个完整使用示例（含输入输出）
